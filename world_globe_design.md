# 🌍 World Globe ページ 設計書

**プロジェクト名：** GuestHouse Dashboard — World ページ  
**ファイル形式：** TSX（React + TypeScript）  
**作成日：** 2026年7月  
**ステータ：** 設計フェーズ

---

## 1. 概要・目的

ゲストハウスの宿泊者（主に外国人旅行者）が、世界各国の文化・言語・観光情報を視覚的に楽しみながら探索できるページ。  
スタッフ向けにも、チェックイン前のアイスブレーク素材として活用できる。

### ユーザーストーリー
- 宿泊者が「自分の国」を地球儀上でクリックし、スタッフに挨拶を教えられる
- 他の国を探索して旅行先の情報を得る
- AIチャットで「次にどの国に行くべきか？」などを相談できる

---

## 2. 技術スタック

| 分類 | 技術 |
|------|------|
| フレームワーク | React + TypeScript（TSX） |
| 3D地球儀 | Three.js（`r128`）CDN経由で動的ロード |
| 地球儀データ | GeoJSON（`natural-earth-vector` / TopoJSON変換済みデータ） |
| 3Dランドマーク | Three.js `BoxGeometry` / `ConeGeometry` / `CylinderGeometry`で手作りモデル |
| AIチャット | Anthropic API（`claude-sonnet-4-6`） |
| 音声再生 | Web Speech API（`SpeechSynthesisUtterance`） |
| スタイリング | Tailwind CSS + インラインスタイル（ダッシュボードのデザイントークンに合わせる） |
| 状態管理 | React `useState` / `useRef` / `useCallback` / `useEffect` |

## 5. 機能詳細

### 5-1. 3D地球儀

#### ビジュアルスタイル
- **イラスト風**：リアルな衛星写真ではなく、絵本・地図帳のような手描き風テクスチャ
- 海：深い青（`#1B5B8A`）にグラデーション、波模様のノイズ
- 陸地：緑系（`#4A8B3F`）にGeoJSONポリゴンで各国を塗り分け
- 国境線：白（`#FFFFFF`）、線幅 `0.8px`、半透明
- 山岳地帯：やや明るい緑
- 砂漠地帯：淡いオレンジベージュ
- 極地：薄い水色

#### インタラクション
| 操作 | 動作 |
|------|------|
| ドラッグ | スムーズに地球儀を回転（慣性あり） |
| スクロール | ズームイン・ズームアウト |
| 国クリック | 該当国にフライ（カメラアニメーション）→ 情報パネル表示 |
| 検索Enter | 検索した国にスムーズフライ |
| ホバー | 国ハイライト（明るいオーバーレイ） + 国名ツールチップ |

#### 地球儀の実装方式
```
Three.js SphereGeometry（半径 1.0, 64×64セグメント）
  └── CanvasTexture（2048×1024px で手描き風描画）
        ├── 海の描画（グラデーション）
        ├── GeoJSON → Canvas パスで陸地・国塗り分け
        └── 国境線の描画
```

#### 国境データ
- ソース：`https://raw.githubusercontent.com/holtzy/D3-graph-gallery/master/DATA/world.geojson`
  または `natural-earth-vector` 110m スケールGeoJSON
- 全世界 195カ国以上対応
- 緯度経度 → UV座標変換でキャンバスに描画

---

### 5-2. 3Dランドマークモデル

地球儀上に主要ランドマークを簡易3Dモデルとして配置（オブジェクト浮遊）。

#### 実装方針
- Three.js プリミティブを組み合わせた手作りモデル
- ランドマーク位置は緯度経度から `latLngToXYZ()` で変換
- モデルは地球表面から少し浮かせて、法線方向に立てる
- マウスホバーでモデルが拡大 + ラベル表示

#### 実装するランドマーク例（初期フェーズ）

| 国 | ランドマーク | 3Dモデル方法 |
|---|---|---|
| 🇯🇵 日本 | 東京スカイツリー | `CylinderGeometry`（先細り） + 先端球 |
| 🇫🇷 フランス | エッフェル塔 | `ConeGeometry` + `BoxGeometry`の格子 |
| 🇪🇬 エジプト | ピラミッド | `ConeGeometry`（4面） |
| 🇮🇳 インド | タージマハル | `SphereGeometry`（ドーム） + `BoxGeometry` |
| 🇨🇳 中国 | 万里の長城 | `BoxGeometry`の連続配置 |
| 🇺🇸 アメリカ | 自由の女神 | `CylinderGeometry` + 手 |
| 🇮🇹 イタリア | コロッセオ | `TorusGeometry` + `CylinderGeometry` |
| 🇧🇷 ブラジル | キリスト像 | `BoxGeometry`（十字） |
| 🇦🇺 オーストラリア | オペラハウス | `SphereGeometry`（部分） |
| 🇬🇷 ギリシャ | パルテノン神殿 | `BoxGeometry`（柱の配列） |

#### ランドマークアニメーション
- 常時ゆっくり上下に浮遊（`Math.sin(t)` オフセット）
- クリックした国のランドマークは明るくハイライト
- 地球の回転に追従

#### 挨拶カード
- 1カード = 1挨拶（テキスト + ローマ字 + 意味）
- 右端に `▶ 再生` ボタン → Web Speech API で音声出力
- 再生中は `⏸` に切り替わりアニメーション

#### 食べ物カード
- 2×2 または 2×3 グリッド
- 大きめ絵文字（48px）+ 料理名

- カードに薄い食材カラー背景（料理ジャンルで色分け）

#### 建物・ランドマーク
- リストカード形式
- 左に絵文字アイコン
- クリックで地球儀上の3Dモデルがズームアップ

#### 豆知識
- `💡` アイコン付きカード
- 3〜4つの豆知識をタブ or スワイプで切り替え
- 例：「世界最長の地下鉄路線」「桜の名所ベスト5」

### 5-4. 検索バー

ページ上部中央に常時表示。

```
[ 🔍  国名を入力...（日本語・英語対応）         ]
```

#### 仕様
- プレースホルダー：`「日本」「France」「Egypt」など`
- リアルタイムサジェスト：入力に応じてドロップダウン表示（国名一覧からフィルタ）
- Enter / クリックで地球儀がその国へスムーズフライ
- 日本語・英語・ローマ字でのマッチング対応
- 検索後、フィールドはクリア

#### フライアニメーション
- 現在のカメラ角度 → 目的地の角度へ `SLERP`（球面線形補間）で補間
- 所要時間：約 1.2秒
- イージング：`easeInOutCubic`

## 6. 国データ構造

### TypeScript型定義

```typescript
interface Greeting {
  text: string;       // 現地語テキスト
  romanized: string;  // ローマ字読み
  meaning: string;    // 日本語での意味
  lang: string;       // Web Speech API言語コード (例: "ja-JP")
}

interface Food {
  name: string;       // 料理名
  emoji: string;      // 絵文字
  description?: string; // 簡易説明
}

interface Landmark {
  name: string;       // 名称
  emoji: string;      // 絵文字
  has3DModel: boolean; // 3Dモデルあり/なし
  lat?: number;       // 3Dモデル配置用緯度
  lng?: number;       // 3Dモデル配置用経度
}

interface CountryData {
  code: string;       // ISO 3166-1 alpha-2 コード
  name: string;       // 日本語国名
  nameEn: string;     // 英語国名
  flag: string;       // 国旗絵文字
  capital: string;    // 首都
  language: string;   // 公用語
  lat: number;        // 国の中心緯度
  lng: number;        // 国の中心経度
  color: string;      // 地球儀上の国の塗り色
  greetings: Greeting[];
  foods: Food[];
  landmarks: Landmark[];
  funFacts: string[]; // 豆知識（3〜5個）
}
```

### 国データ収録数目標
- フェーズ1：主要50カ国（初期リリース）
- フェーズ2：全195カ国（段階的追加）

## 10. パフォーマンス考慮事項

| 課題 | 対策 |
|------|------|
| GeoJSON描画の重さ | テクスチャは初回1回のみ生成・キャッシュ |
| 3Dランドマーク数 | 視野外のランドマークはレンダリング対象外（frustum culling） |
| APIコスト | AIチャットはデバウンス処理 + 1回につきmax_tokens: 400 |
| モバイル対応 | canvas解像度をdevicePixelRatio × 0.75で制限 |
| Three.js CDN | `r128` を固定バージョン指定（breaking changes防止） |

## 5-6. スペシャルエフェクト（ポップ演出）

地球儀操作時に楽しい視覚効果を追加。

### 実装するエフェクト

| エフェクト名 | トリガー | 説明 |
|-------------|---------|------|
| 🎊 コンフェッティ | 国クリック/到着 | カラフルな紙吹雪が画面全体に舞う（Canvas パーティクル） |
| ✨ スパークル | ランドマークホバー | きらきら星が瞬く（Sprite アニメーション） |
| 🌊 波紋 | 海エリアホバー | 同心円状の波紋が広がる（シェーダー） |
| 💫 グロー | 国ホバー | 国境が光る（Post-processing Bloom） |
| 🎈 バルーン浮遊 | 待機時（10 秒） | 小さな風船が画面下から浮かび上がる |
| 🌟 ジャンプ | ダブルクリック | ランドマークが大きくジャンプ |
| 🌀 回転フィードバック | ドラッグ終了 | 慣性回転終了時に少しバウンド |

### 実装技術
- パーティクルシステム：Three.js `Points` + `ShaderMaterial`
- ポストプロセッシング：`EffectComposer` + `UnrealBloomPass`
- CSS アニメーション：UI エレメント用
