**作成日：** 2026年7月  
**ステータス：** 設計フェーズ（v2 — AIチャット・3Dランドマーク機能を除外）

## 1. 概要・目的

ゲストハウスの宿泊者（主に外国人旅行者）が、世界各国の文化・言語・観光情報を視覚的に楽しみながら探索できるページ。  
スタッフ向けにも、チェックイン時のアイスブレーク素材として活用できる。

### ユーザーストーリー
- 宿泊者が「自分の国」を地球儀上でクリックし、挨拶や文化を紹介し合える
- 他国を探索して、旅行先の情報を得る
- 検索バーで国名を打ち込み、地球儀がその国にスムーズに飛ぶ

### スコープ（v2）
| 機能 | 状態 |
|------|------|
| イラスト風クレイ調3D地球儀 | ✅ 実装する |

## 4. 地球儀ビジュアル詳細設計

### 4-1. 質感：クレイ調（Clay Morphism）

リアルな衛星写真ではなく、**粘土細工のような柔らかく立体的なイラスト風**を目指す。

| 要素 | 実装方法 |
|------|----------|
| 球体の質感 | `MeshPhongMaterial` — `shininess: 18`、`specular: #C0D8F0`（やや光沢） |
| 環境光 | `AmbientLight` 色`#FFF8F0`、強度 `0.6` |
| メインライト | `DirectionalLight` 色`#FFFFFF`、強度 `1.1`、位置 `(4, 3, 5)`（右手前上） |
| フィルライト | `DirectionalLight` 色`#D0E8FF`、強度 `0.3`、位置 `(-3, -1, -2)`（左奥下） |
| 大気圏グロー | `SphereGeometry` 半径 `1.015`、`MeshPhongMaterial` color `#A8D8FF`、transparent `true`、opacity `0.06`、side `FrontSide` |
| ドロップシャドウ | 地球儀下に楕円形の `PlaneGeometry` + グラデーションテクスチャ、または CSS `filter: drop-shadow` |

### 4-2. テクスチャ生成（Canvas描画）

OffscreenCanvas（`2048 × 1024 px`）にGeoJSONを使って手描き風に描画し、`THREE.CanvasTexture`に変換する。

```
描画順序：
  1. 海の塗りつぶし（ライトブルーグラデーション）
  2. GeoJSONの各国ポリゴンを緯度経度 → UV座標変換で描画
     └── 高度データに応じた色分け（平野: 黄緑、砂漠: ベージュ、極地: 白青）
  3. 国境線を白で上描き（opacity 0.7）
  4. 緯度経度グリッドライン（淡い水色、opacity 0.1）
  5. テクスチャにソフトなノイズを重ねてクレイ感を演出
```

**緯度経度 → UV座標の変換式：**
```typescript
const u = (lng + 180) / 360;
const v = (90 - lat) / 180;
const x = u * canvasWidth;
const y = v * canvasHeight;
```

### 4-3. ライティングによるクレイ調演出

```typescript
// メインライト（右手前から柔らかく照らす）
const sunLight = new THREE.DirectionalLight(0xFFFFFF, 1.1);
sunLight.position.set(4, 3, 5);
scene.add(sunLight);

// フィルライト（左奥から薄く補光）
const fillLight = new THREE.DirectionalLight(0xD0E8FF, 0.3);
fillLight.position.set(-3, -1, -2);
scene.add(fillLight);

// 環境光（全体の柔らかな底上げ）
const ambientLight = new THREE.AmbientLight(0xFFF8F0, 0.6);
scene.add(ambientLight);

// 地球儀マテリアル
const globeMaterial = new THREE.MeshPhongMaterial({
  map: canvasTexture,
  specular: new THREE.Color(0xC0D8F0),
  shininess: 18,
  bumpScale: 0.005,  // テクスチャのバンプ感（任意）
});
```

### 4-4. フローティング感の演出

地球儀が空中に浮かんでいる視覚効果を、以下2つで実現する。

**① CSSドロップシャドウ（canvasタグ）：**
```css
canvas {
  filter: drop-shadow(0 40px 60px rgba(100, 120, 180, 0.18))
          drop-shadow(0 12px 20px rgba(100, 120, 180, 0.10));
}
```

**② Three.js内の影楕円（任意）：**
```typescript
// 地球儀直下に楕円形の影を配置
const shadowGeo = new THREE.PlaneGeometry(2.2, 0.6);
const shadowMat = new THREE.MeshBasicMaterial({
  color: 0x8899BB,
  transparent: true,
  opacity: 0.12,
});
const shadowMesh = new THREE.Mesh(shadowGeo, shadowMat);
shadowMesh.position.set(0, -1.2, 0);
shadowMesh.rotation.x = -Math.PI / 2;
scene.add(shadowMesh);
```

## 9. 国データ構造（TypeScript型定義）

```typescript
interface Greeting {
  text: string;        // 現地語テキスト
  romanized: string;   // ローマ字読み
  meaning: string;     // 日本語での意味
  lang: string;        // Web Speech API言語コード（例: "ja-JP"）
}

interface Food {
  name: string;        // 料理名（日本語）
  emoji: string;       // 絵文字
  category?: string;   // カテゴリ（和食 / 洋食 / アジア / etc）
}

interface LandmarkInfo {
  name: string;        // 建物・場所名（日本語）
  emoji: string;       // 絵文字
}

interface CountryData {
  code: string;        // ISO 3166-1 alpha-2（例: "JP"）
  name: string;        // 日本語国名
  nameEn: string;      // 英語国名
  flag: string;        // 国旗絵文字
  capital: string;     // 首都（日本語）
  language: string;    // 公用語
  lat: number;         // 国の中心緯度（ピン・フライ先）
  lng: number;         // 国の中心経度
  color: string;       // ピンカラー（hex）
  greetings: Greeting[];
  foods: Food[];
  landmarks: LandmarkInfo[];
  funFacts: string[];  // 豆知識（3〜5個）
}
```

## 10. インタラクション仕様まとめ

| 操作 | 動作 |
|------|------|
| 地球儀ドラッグ | 慣性付き回転（`rotX` / `rotY` の減衰） |
| スクロール（ホイール） | カメラズームイン・アウト（`camera.position.z`を変化） |
| 国ピン ホバー | ポップアップカード表示（200ms フェードイン） |
| 国ピン クリック | ① 地球儀がその国へSLERPフライ（1.2秒）、② 情報パネルスライドイン |
| 地球儀の国ポリゴン クリック | Raycasterで国判定 → 上記と同じ動作 |
| 検索バー 入力 | リアルタイムサジェスト表示 |
| 検索バー Enter / サジェスト選択 | 地球儀フライ + 情報パネル表示 |
| パネル `×` ボタン | 情報パネルスライドアウト |
| `＋` / `−` ボタン | ズームイン・アウト（50ms ごとに段階的に） |
| コンパスボタン | 地球儀の傾きをデフォルト角度にリセット（アニメーション付き） |
| 再生ボタン | Web Speech APIで挨拶を音声再生。再生中はボタンが⏸ |

## 14. パフォーマンス考慮事項

| 課題 | 対策 |
|------|------|
| GeoJSON描画コスト | テクスチャは初回1回のみ生成、`useMemo`でキャッシュ |
| ピン多数のDOM | 表示範囲内（地球の表側）のみレンダリング（`dot product`で裏側を除外） |
| テクスチャ解像度 | モバイルは `1024×512px`、デスクトップは `2048×1024px` |
| Three.js CDN読み込み | `script.onload` を待ってから初期化（動的ロード） |
| アニメーションFPS | `requestAnimationFrame` + delta time 管理で60fps維持 |
